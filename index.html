What is Smash Browse?

Smash Browse saves you from daily online chores by automating your Chrome web browser.

Other Web Automation Tools blindly record your web actions, often identifying HTML elements explicitly by using XPath identifiers, CSS selectors, or element IDs. This technique is not robust as the identifiers soon become out of date when the web page is updated with different element identifiers.

Smash Browse uses a new technique to capture more of the semantic meaning and intent behind your actions. Smash Browse identifies elements using only semantic information about the element, such as its text, and by using constraints to locate elements, such as its proximity to other elements. Because of this, Smash Browse automations are not tightly coupled to the implementation (XPath, CSS, IDs) of the web page.

Nodes

A node represents a task that Smash Browse will perform for you. A node will perform some type of computation on its inputs in order to produce some outputs. For example, a script node could add two numbers together. In this case, the inputs are the two numbers and the output is the sum. (1+1=2)

The name of the node appears directly on the node. In this case, we see that the name is "data".

A node shows its inputs as boxes and its outputs as upside down triangles. The data types that flow into an input or flow out of an output are modeled after javascript data types. The complete set of data types are numbers, strings, booleans, objects, and arrays.


Editing Inputs

When you edit a node, you are editing the default values of the inputs on that node. The default value plays an important role in determining the final value of the input.

When editing a node, you can also expose or hide any of the inputs. When an input is exposed, a link can direct data from an upstream node into the input.

The current node being edited will be marked on the right with the letter "E".

These are the rules that determine how the default value affects the final value of the input.

  • If the input is not linked to any other node, then the default value is the final value of the input.
  • If the input is linked to an output on an upstream node, then the upstream node's value is merged into the default value to produce the final value of the input. The merging process is described further in the section on links below.

Cleaning Nodes

Cleaning a node forces the node to perform its computation. A computation is an operation where it uses its input values to compute its output values.

In order for a node to output a clean (up to date) value, its inputs must be clean as well. Hence when you a clean a node, all of the linked upstream nodes will be cleaned recursively as well.

After a node is cleaned it will be marked on the right with the letter "C". When a node is dirty, the letter "C" will be absent.


Viewing Outputs

After a node is cleaned, you can view the latest output values produced by the node. Nodes can output any of the usual data types: numbers, strings, boolean, objects, and arrays. The user interface will allow you to dive into objects and arrays by double clicking on them.

You can also view the outputs of a dirty node. However, the values will no longer be current or up to date.

The current node being viewed will be marked on the right with the letter "V".


Links

A link is used to direct output data from an output on an upstream node to an input on a downstream node. The data flowing through the link will be merged into the input's default value to produce the final value. Before merging, the type and internal structure of the upstream data is converted to the same type and internal structure of the default value. After which any internal properties in the default value with the same name as the properties in the upstream data get overridden with the upstream value. If the default value does not contain a corresponding property from the upstream data, it will be added in.


Groups

A group node is a node which contains another node graph inside itself. When you dive into a group node, you will see the contents of the group node. When you surface from a group you will see the node graph in which the group node exists.

Group nodes allow you to collapse a complex node graph into one node and to also define a strict interface to the logic contained within. The interface is represented by the inputs and outputs on the group node.

In order to create inputs or outputs on a group node, you will first need to dive into the group. Then create Input Nodes or Output Nodes respectively. Each Input Node or Output Node inside the group will appear as inputs or outputs respectively on the group node under the same names.

There are a few different types of group nodes. Each provides a specialized context for their nodes to operate in. They are data group nodes, script group nodes, and web group nodes.


Browser Groups

The Browser Group Node provides a browser management context for nodes which manipulate the browser. When diving into a Browser Group Node, a browser window will open automatically. When surfacing from inside a Browser Group Node, the browser window will close automatically. All of the internal nodes can make use of this browser.

Unlike other nodes, to create nodes which use the browser you will need to right-click inside the browser. This will bring up a menu containing possible nodes to create. The position of the right-click in the browser will be used to automatically set a number of input values on the nodes it creates. See the section on Automating the Browser for more details.


Posts

A Post Node allows you to extract some data that is flowing through a link and to post that data to a page inside of Smash Browse. This is similar to posting a status update on social media, except it's for yourself and only you can see it.

This can be used to verify that your automation completed successfully or to notify you when certain events of importance occur inside your node graph.

For example, you can post bank account balances, recent price changes on items you're interested in, recent news posts or social media updates, etc.


JavaScript

Javascript is the language used to specify the script for our script nodes. Script nodes are very powerful, allowing users to essentially create brand new node types.

A script node only contains one input and one output. They are accessible in the scripting environment as "input" and as "output".

In the following example, the output's value property is being set to the sum of the input's x and y properties.


        output.value = input.x + input.y;

When the script node is placed inside a "while" node or another looping node, the script will often need to share state between iterations. The scripting environment provides a context object which will hold any properties between iterations. In the following example, the script node counts how many times it loops and sets it as the output's value property. 


        if (context.count == undefined) {
            context.count = 1
        } else {
            context.count += 1
        }
        output.value = context.count

Automating the Browser

These are the basic steps to automating the browser. Watching these video demos will likely make things much clearer.

  • To control the browser you'll first need to create a Browser Group Node. Then you'll need to dive into it, to create browser automation nodes.

  • Once you dive into the Browser Group you'll see a browser window open up automatically. Please wait while it loads the first web page.

  • We are now ready to create nodes. Unlike the other groups where you right-click in the node graph to bring up a menu to create nodes when you're inside a Browser Group you right-click in the browser.

  • The position of the right-click is significant as it allows Smash Browse to automatically determine and set the input values on the nodes that it creates for you. This makes the creation of nodes much quicker. Otherwise, you'd need to manually edit each node to set appropriate values for the behavior you want, by yourself.

  • Note that newly created nodes will automatically be connected to the last selected node. This reduces the amount of time you'll spend linking up nodes. So make sure you select the node you want to link to before creating a node.

  • There are 3 basic steps to performing actions in the browser.

    • First, you need to create a node which outputs data containing an array of elements in the browser or identify an upstream node which outputs the same. This array can contain just one element or it can contain many elements. An element in Smash Browse is either an image, a piece of text, an input field, or a select dropdown. Note that you can visualize the elements by feeding it into the Highlight Elements node.

    • Second, you will often need to isolate the elements you're actually interested in from the rest of the elements in the array of elements. To do this you can use any of the nodes which perform filtering operations on the array of elements. Isolate, prune, sort, join and remove are some common terms used in the names of these nodes.

    • Third, you need to feed these elements into a node which performs a browser action. Many nodes will only act on the first element in the array elements. For example, the MouseAction node will only click the first element. Other nodes like the download videos node will act on all the input elements.

  • By following the above tips, you will soon find yourself building extremely useful and powerful node graphs.
